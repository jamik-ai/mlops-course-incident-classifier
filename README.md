# MLOps проект: прогнозирование количества вызовов

## Описание проекта
Данный проект реализует MLOps-систему для прогнозирования количества вызовов в час на следующий день.  
На основе исторических данных о звонках (дата/время, тип звонка, значение Kp) мы строим модель временного ряда, которая предсказывает загрузку системы по часам.

Проект охватывает начальные этапы MLOps-пайплайна:
- Версионирование данных (DVC)
- Очистка и предобработка данных
- Формирование признаков для временного ряда
- Обучение модели (XGBoost) с оценкой качества (MAE, RMSE)
- Сохранение артефактов (модель, признаки)

В дальнейшем планируется добавить:
- FastAPI сервис для получения прогнозов
- Веб-интерфейс (страница ввода даты, отображение прогноза)
- Мониторинг дрейфа данных (Prometheus + Grafana)
- CI/CD (GitHub Actions)
- Деплой в Kubernetes (minikube) с ArgoCD
